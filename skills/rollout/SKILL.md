## Your Task

Produce `.patchline/artifacts/ROLLOUT.md`, a dated rollout calendar that tells the artist what content gets made, where it goes, what outreach happens, and who owns it.

Do not turn rollout into a long intake form. Read release-plan and artist context first; social-channel planning should happen automatically from known data or sensible baseline assumptions.

## Step 1: Read Workspace Context

Use Read:

- `.patchline/PROJECT.md` - artist identity, Project Anchor ID, campaign intake.
- `.patchline/STATE.md` - confirm `Current phase: rollout`.
- `.patchline/artifacts/RELEASE_PLAN.md` - release date, playlist targets, DSP priority, promo tier, distribution action items.
- `.patchline/artifacts/VISION.md` - tone and story.
- `.patchline/artifacts/MOODBOARD.md` - visual mood and anchor track.
- `.patchline/artifacts/BRIEF.md` - audience, north-star metric, non-negotiables.

If ROLLOUT.md already exists, ask whether to overwrite or refine.

## Step 2: Pull Artist Context

Call `mcp__aria__get_artist_context` with artist/project identifiers from PROJECT.md.

Use it to set:

- Social channels: use connected/social-intelligence data automatically when available.
- If social data is unknown, default to a baseline multi-channel rollout: Instagram, TikTok, YouTube Shorts, and Spotify/Apple artist-profile actions. Mark performance data as unknown; do not ask which socials the artist is "active on."
- Email list: returned bucket, otherwise `none`.
- Distributor/team context: if unknown, owner defaults to Artist.
- Telegram linked: if true, note that Telegram can be used for reminders/follow-ups later.

Do not ask the artist to self-report social-channel activity as a gate. Patchline should use known context automatically or default the campaign around baseline channels. If they later correct the channel mix, respect the correction.

## Step 3: Optional MCP Check

Optionally call `mcp__aria__generate_pitch` with the real 3-arg signature (`artist_name`, optional `track_name`, optional `target_playlist`) for one top target from RELEASE_PLAN.md. Treat this as a pitch-shape check only; final pitches are generated in pitch-kit.

If it errors, note the error and continue.

## Step 4: Ask Only Execution-Critical Questions

Ask one compact content-capacity question:

> How much content can you realistically make before release? Give numbers for photos, short vertical clips, and long-form video, or say "lean".

If RELEASE_PLAN.md has promo tier, primary social, email bucket, and date, do not ask more. If any are missing:

- Promo tier: ask with `AskUserQuestion`.
- Release date: ask for date or TBD.
- Team owner: default to `artist solo`; ask only if label mode or context suggests a team.

Never ask social-channel activity as a required question.

## Step 5: Draft ROLLOUT.md

Use Write:

```markdown
# Rollout Calendar: <project-name>

> Generated by `/aria:rollout` on YYYY-MM-DD - v1
> Grounded in: RELEASE_PLAN.md, VISION.md, MOODBOARD.md, BRIEF.md, get_artist_context, optional generate_pitch preview, user content-capacity answer.

## Assumptions

- Release date: <date or TBD>
- Social channels: <known connected/social-intelligence channels, or baseline Instagram + TikTok + YouTube Shorts + artist-profile actions>
- Social data confidence: <known | partial | unknown>
- Email list: <bucket or none>
- Promo tier: <none | modest | significant>
- Execution owner: <artist solo | artist + manager | label-coordinated>
- Content capacity: <photos / clips / long-form>

## Calendar

| Week | Date | Content | Channel | Action | Owner |
|---|---|---|---|---|---|
| -6w | <date or TBD> | Announce / project context | Primary social | Publish/pin | Artist |
| -4w | <date or TBD> | Editorial pitch submission | DSP portal/distributor | Submit | Artist/label |
| -3w | <date or TBD> | Visual reveal | Primary social | Publish | Artist |
| -2w | <date or TBD> | Playlist/curator outreach starts | Email/DM/forms | Send pitch-kit drafts | Artist/manager |
| -1w | <date or TBD> | Pre-save / smart-link prep | Primary social | Publish CTA when link exists | Artist |
| 0 | <release date> | Release-day post | Baseline social channels + artist profiles | Smart-link push | Artist |
| +1w | <date or TBD> | Analytics check + second-wave outreach | DSP dashboards + DM/email | Review and follow up | Artist/manager |
| +4w | <date or TBD> | Campaign retrospective | Patchline / notes | Compare to north-star metric | Artist |

## Content Plan

<Match recommendations to capacity. If the user said lean, plan one strong photo, one short clip, and one release-day post instead of a huge cadence.>

## Outreach Waves

- Editorial: <from release-plan>
- Curated playlists: <from release-plan tiers>
- Niche/independent: <from release-plan tiers>
- Follow-up wave: <+1w>

## Risks And Contingencies

- Content delay: <what to drop first>
- Distributor/submit deadline: <from release-plan>
- Playlist verification gaps: <from release-plan>
- Email/social unknown assumptions: <state assumptions plainly>

## Data Sources

- Artist context: <social data known/partial/unknown, email/distributor/Telegram known or unknown>
- Release plan: <date, promo tier, targets>
- User content capacity: <answer>
- Pitch preview: <ran / skipped / errored>
```

## Step 6: Update STATE.md

Append only:

- `- rollout - completed YYYY-MM-DD, artifact: ROLLOUT.md`
- `- ROLLOUT.md (rollout phase, generated YYYY-MM-DD)`
- `Last updated: YYYY-MM-DD by /aria:rollout`

Do not touch `Current phase`.

## Step 7: Hand Off

Tell the user in three sentences or fewer that ROLLOUT.md is written, state the social-data assumption and content-capacity assumption, and ask them to run `/aria:next` for pitch-kit.

## Error Handling

- If `get_artist_context` fails, continue with baseline multi-channel rollout assumptions and mark social data confidence as unknown.
- If RELEASE_PLAN.md is missing or incomplete, stop and tell the user to run `/aria:next` so the lifecycle can route correctly.
- If `generate_pitch` fails, record the preview error and continue; pitch-kit owns final pitch generation.
- If the user gives vague content capacity, default to a lean plan rather than asking a long follow-up interview.

## Examples

Known social context: use the connected platforms automatically and do not ask which socials the artist is active on.

Unknown social context: plan Instagram, TikTok, YouTube Shorts, and artist-profile actions as the baseline, with social data confidence marked unknown.

## Common Mistakes

- Asking known or defaultable context questions, especially social-channel activity.
- Planning more content than the user can realistically make.
- Creating new playlist targets not in RELEASE_PLAN.md.
- Treating social/email unknowns as blockers.
