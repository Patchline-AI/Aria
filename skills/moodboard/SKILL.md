## Your Task

Produce `.patchline/artifacts/MOODBOARD.md`, a working moodboard that turns VISION.md into concrete sonic targets, visual direction, and production cues.

If `Focus track asset ID` exists, that focus track is always the anchor. Do not ask the user which tracks should anchor the moodboard. Catalog tracks are supplementary references only: they can color, contrast, or sharpen the focus track.

Every BPM, key, energy score, valence score, and mood tag must come from MCP output or AUDIO_INTAKE.md. No invented numbers.

## Step 1: Read Workspace Context

Use Read in this order:

- `.patchline/STATE.md` - confirm `Current phase: moodboard`; capture `Composition status`, `Focus track asset ID`, and `Track analysis status`.
- `.patchline/PROJECT.md` - extract artist name, Project Anchor ID, primary genres, distribution mode, and campaign intake.
- `.patchline/artifacts/AUDIO_INTAKE.md` - if present, extract focus track, asset ID, audio features, visual references, desired assets, and campaign intake.
- `.patchline/artifacts/BRIEF.md` - intent, audience, success metric.
- `.patchline/artifacts/VISION.md` - sonic identity, story, tonal palette.
- `.patchline/artifacts/MOODBOARD.md` - if it exists, ask whether to overwrite, refine, or cancel.

If VISION.md is missing, stop and tell the user to run `/aria:next`.

## Step 2: Resolve The Focus Track

If `Focus track asset ID` is neither `pending` nor `not_required`:

1. Call `mcp__aria__get_asset` for the focus track.
2. Call `mcp__aria__get_audio_features` if needed for thinner feature projection.
3. Treat that focus track as the anchor. Its BPM/key/genres/moods become the core target.

If no focus track exists, fall back to catalog/reference selection as described below and be honest that there is no uploaded-track anchor.

## Step 3: Find Supplementary References

Supplementary references are optional. They should help answer: "What should color, contrast, or sharpen the focus track?"

Derive two queries from VISION.md and AUDIO_INTAKE.md:

- Vibe query: 2-3 tonal words plus one sonic phrase.
- Genre query: one primary genre from PROJECT.md or the focus-track genre.

Call `mcp__aria__catalog_search` with the vibe query, limit 10. If fewer than 3 relevant hits appear, call it again with the genre query. If both are thin, call `mcp__aria__browse_catalog` with `{ page: 1, pageSize: 20 }`.

For up to 5 candidates, call `mcp__aria__get_song_intelligence` with `{ assetId }`; if that errors, try `mcp__aria__get_audio_features`. Mark unanalyzed candidates clearly and do not use them for numeric ranges.

If a focus track exists, ask at most one optional selection question:

> I am anchoring the moodboard on your uploaded track. Which catalog references, if any, should color or contrast it?

Use `AskUserQuestion` only for supplementary references. The question must not include "which tracks should anchor". Include a "No supplementary references" option.

If no focus track exists, ask which analyzed tracks should serve as references, but still use "references", not "anchors", unless the user explicitly chooses one as the anchor.

## Step 4: Capture Visual Intake

If AUDIO_INTAKE.md / PROJECT.md already contains visual references, confirm with one sentence and do not re-ask unless unclear.

Otherwise ask:

> Any visual references for this release? Paste image paths, URLs, mood words, or say no visuals yet.

Then ask one optional production cue question:

> Any production cues you want to lean into? Reverbs, rooms, processing, arrangement quirks, or say no specific cues.

## Step 5: Build Feature Targets

If focus-track features exist:

- Start BPM/key/genre/mood from the focus track.
- Use supplementary analyzed references only to widen or qualify the range.
- If supplementary references conflict with the focus track, label them as contrast references rather than changing the anchor.

If no focus track exists:

- Build ranges from analyzed confirmed references only.
- If no analyzed references exist, skip numeric feature targets.

## Step 6: Draft MOODBOARD.md

Use Write:

```markdown
# Moodboard: <project-name>

> Generated by `/aria:moodboard` on YYYY-MM-DD - v1
> Grounded in: BRIEF.md, VISION.md, AUDIO_INTAKE.md / focus-track analysis if present, catalog_search, get_song_intelligence/get_audio_features, user confirmation.

## Anchor track

- Track: <focus track title or "none selected">
- Asset ID: <assetId or none>
- BPM/key: <real values or unavailable>
- Core read: <genres/moods/energy/valence summary>

## Supplementary references

| Track | Role | BPM | Key | Energy | Valence | Source |
|---|---|---|---|---|---|---|
| <title> | color / contrast / sharpen | <real> | <real> | <real> | <real> | User catalog |

If none: "No supplementary catalog references selected; the focus track carries the moodboard."

## Audio-feature targets

<If focus track exists: anchor-centered target range.>
<If only references exist: reference-derived range.>
<If no analyzed audio exists: state that no numeric feature targets were set.>

## Visual mood

<Image paths/URLs/mood words or "no visuals yet".>

## Production cues

<User cues or "no specific production cues flagged".>

## What this moodboard does NOT cover

- Release schedule and playlist targets - release-plan phase.
- Cover/flyer/canvas generation - campaign tooling; this moodboard only captures direction.

## Data sources

- Focus track: <assetId + analysis status>
- Supplementary references: <N searched, N analyzed, N selected>
- User visual intake: <date>
```

Use "track analysis" in user-facing prose. Keep vendor-specific raw field names only if quoting an internal field in Data sources.

## Step 7: Update STATE.md

Append only:

- `- moodboard - completed YYYY-MM-DD, artifact: MOODBOARD.md`
- `- MOODBOARD.md (moodboard phase, generated YYYY-MM-DD)`
- `Last updated: YYYY-MM-DD by /aria:moodboard`

Do not touch `Current phase`.

## Step 8: Hand Off

Tell the user the moodboard is written, name the anchor track, mention whether supplementary references were added, and ask them to run `/aria:next`. If `Composition status` is complete, note that songwriting-brief is skipped and release-plan is next.

## Error Handling

- If the focus-track asset cannot be loaded, use AUDIO_INTAKE.md as the cached source and clearly mark live asset lookup unavailable.
- If catalog search returns no useful supplementary tracks, continue with the focus track as the sole anchor.
- If visual references are invalid paths/URLs, keep the mood words and note that files were not verified.
- If the user declines visual intake, write `no visuals yet` rather than blocking.

## Examples

Finished single with focus asset: write the focus track as the anchor, use catalog tracks only as optional color/contrast/sharpen references, and include one visual-intake answer.

Idea-first project without focus asset: build the moodboard from confirmed catalog references and user mood words, with no numeric claims unless MCP data supports them.

## Common Mistakes

- Asking "Which tracks should anchor the moodboard?" when a focus track exists.
- Replacing the uploaded track with a catalog reference.
- Inventing feature ranges.
- Skipping visual intake.
- Exposing vendor names as product copy.
